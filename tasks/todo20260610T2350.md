## Review (completed 2026-06-11)

Five parallel adversarial agents swept AEAD/KDF, ECC, key-model/JWK, BBS/FFI, and
signers/DI/CI. Every finding was re-verified against source before fixing. Result:
**12 fixes applied + 3 design items flagged**, suite 683 native / 664 no-native, 10
samples exit 0, API coverage clean, both workflows valid.

### Fixed (NFR-3 input-validation ŌĆö the L1/L3 crash class, in new locations)
1. `ConcatKdf.DeriveKey` ŌĆö oversized `keyDataLen` ŌåÆ `OverflowException`. Added upper-bound
   guard mirroring `Hkdf`. (ConcatKdf.cs)
2. `KeyTypeExtensions.NormalizeToCompressed(null)` ŌåÆ `NullReferenceException`. Added
   `ThrowIfNull`. (KeyTypeExtensions.cs)
3. `JwkConverter.ExtractPublicKey` ŌĆö raw `FormatException` from `Multibase.Decode` on bad
   base64url (CidFormatException : FormatException). Wrapped all 3 decodes ŌåÆ ArgumentException.
4. `JwkConverter.ExtractPublicKey` ŌĆö left-trimmed EC `x` produced a short SEC1 point (32 vs 33).
   Now left-pads to the curve field width; round-trips.
5. `InMemoryKeyStore` ŌĆö `ImportAsync(alias, null)` ŌåÆ NRE; null/empty alias unguarded. Added
   `ThrowIfNullOrEmpty(alias)` / `ThrowIfNull(keyPair)` to all six public methods.
6. `DefaultBbsCryptoProvider.DeriveProof` ŌĆö out-of-range/negative `revealedIndices` cast to a
   huge uint at the FFI ŌåÆ opaque CryptographicException. Added range validation ŌåÆ
   ArgumentOutOfRangeException.
7. `DefaultKeyGenerator.DeriveX25519FromEd25519` ŌĆö null pair / wrong-length seed ŌåÆ NRE /
   raw NSec FormatException. Added null + 32-byte guards (matching the public-only sibling).
8. `DefaultKeyGenerator.DeriveX25519PublicKeyFromEd25519` ŌĆö `y == 1` silently produced a
   degenerate `u = 0`. Now rejects (birational map undefined there).

### Fixed (contract / correctness)
9. `DefaultCryptoProvider.Verify` (NIST) ŌĆö an off-curve but parseable public key threw
   `CryptographicException` instead of returning `false` (the import sat *outside* the try).
   Moved import inside the try; off-curve key ŌåÆ `false`, JWS-loop-safe. (Wrong-length/format
   still throws ArgumentException per NFR-3.)

### Fixed (memory safety / hardening)
10. Rust FFI (`lib.rs`) ŌĆö a panic inside zkryptium would unwind across `extern "C"` into the
    .NET runtime (UB). Added a centralized `catch_unwind` guard (`ffi_call`) on all six exports;
    panics now map to the existing `-1` error path. (Also removed a pre-existing unused-var
    warning in a Rust test.)
11. `AesKeyWrap` ŌĆö widened the `t = n*j` round counter to 64-bit (latent int overflow at ~4 GB;
    unreachable but hygienic). Both Wrap and Unwrap.
12. `Secp256k1Recoverable.RecoverPublicKey` ŌĆö doc said "canonical" but high-S (malleable)
    signatures are accepted (correct for a low-level recovery primitive). Clarified the remark
    so callers know to enforce EIP-2 low-S themselves; enforcement intentionally unchanged.

### CI / supply-chain hardening
- `build.yml`: added `permissions: contents: read` (was default read/write on PRs); `--locked`
  on cargo build/test.
- `release.yml`: dropped workflow-wide `contents: write` to `read`, granted `contents: write`
  only to the `publish` job; `--locked` on the native build.
- `build-all.sh`: `--locked`.

### Design changes (maintainer-approved 2026-06-11, second commit)
- **Defensive copies**: `KeyPair.PublicKey/PrivateKey`, `PublicKeyReference.PublicKey`,
  `StoredKeyInfo.PublicKey` now clone on init (with null guard) and on get; internals read the
  backing field directly. Public signatures unchanged (byte[] preserved ŌåÆ no API-surface change).
- **`StoredKeyInfo` value equality**: custom `Equals(StoredKeyInfo?)`/`GetHashCode` compare
  Alias + KeyType + PublicKey *content* (HashCode.AddBytes). Two PublicAPI.Shipped.txt entries
  added. Coverage tool already exempts Equals/GetHashCode (inherited-names list).
- **`FromPublicKey` validation**: rejects non-canonical lengths (`IsValidKeyLength`) and
  off-curve SEC1 points (`IsValidEcPoint`) with ArgumentException(paramName: publicKey).
  Canonical-only contract documented on IKeyGenerator + taught in the Keys sample;
  uncompressed points route via `NormalizeToCompressed` (round-trip regression-tested).
  Deliberate deviation from verbatim FR-6 parity, per lessons L1 (NFR wins on invalid input).
- 11 regression tests added. Suite after design changes: **694 native / 675 no-native**, 10
  samples exit 0, coverage clean.
