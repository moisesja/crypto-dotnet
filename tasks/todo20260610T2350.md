## Review (completed 2026-06-11)

Five parallel adversarial agents swept AEAD/KDF, ECC, key-model/JWK, BBS/FFI, and
signers/DI/CI. Every finding was re-verified against source before fixing. Result:
**12 fixes applied + 3 design items flagged**, suite 683 native / 664 no-native, 10
samples exit 0, API coverage clean, both workflows valid.

### Fixed (NFR-3 input-validation — the L1/L3 crash class, in new locations)
1. `ConcatKdf.DeriveKey` — oversized `keyDataLen` → `OverflowException`. Added upper-bound
   guard mirroring `Hkdf`. (ConcatKdf.cs)
2. `KeyTypeExtensions.NormalizeToCompressed(null)` → `NullReferenceException`. Added
   `ThrowIfNull`. (KeyTypeExtensions.cs)
3. `JwkConverter.ExtractPublicKey` — raw `FormatException` from `Multibase.Decode` on bad
   base64url (CidFormatException : FormatException). Wrapped all 3 decodes → ArgumentException.
4. `JwkConverter.ExtractPublicKey` — left-trimmed EC `x` produced a short SEC1 point (32 vs 33).
   Now left-pads to the curve field width; round-trips.
5. `InMemoryKeyStore` — `ImportAsync(alias, null)` → NRE; null/empty alias unguarded. Added
   `ThrowIfNullOrEmpty(alias)` / `ThrowIfNull(keyPair)` to all six public methods.
6. `DefaultBbsCryptoProvider.DeriveProof` — out-of-range/negative `revealedIndices` cast to a
   huge uint at the FFI → opaque CryptographicException. Added range validation →
   ArgumentOutOfRangeException.
7. `DefaultKeyGenerator.DeriveX25519FromEd25519` — null pair / wrong-length seed → NRE /
   raw NSec FormatException. Added null + 32-byte guards (matching the public-only sibling).
8. `DefaultKeyGenerator.DeriveX25519PublicKeyFromEd25519` — `y == 1` silently produced a
   degenerate `u = 0`. Now rejects (birational map undefined there).

### Fixed (contract / correctness)
9. `DefaultCryptoProvider.Verify` (NIST) — an off-curve but parseable public key threw
   `CryptographicException` instead of returning `false` (the import sat *outside* the try).
   Moved import inside the try; off-curve key → `false`, JWS-loop-safe. (Wrong-length/format
   still throws ArgumentException per NFR-3.)

### Fixed (memory safety / hardening)
10. Rust FFI (`lib.rs`) — a panic inside zkryptium would unwind across `extern "C"` into the
    .NET runtime (UB). Added a centralized `catch_unwind` guard (`ffi_call`) on all six exports;
    panics now map to the existing `-1` error path. (Also removed a pre-existing unused-var
    warning in a Rust test.)
11. `AesKeyWrap` — widened the `t = n*j` round counter to 64-bit (latent int overflow at ~4 GB;
    unreachable but hygienic). Both Wrap and Unwrap.
12. `Secp256k1Recoverable.RecoverPublicKey` — doc said "canonical" but high-S (malleable)
    signatures are accepted (correct for a low-level recovery primitive). Clarified the remark
    so callers know to enforce EIP-2 low-S themselves; enforcement intentionally unchanged.

### CI / supply-chain hardening
- `build.yml`: added `permissions: contents: read` (was default read/write on PRs); `--locked`
  on cargo build/test.
- `release.yml`: dropped workflow-wide `contents: write` to `read`, granted `contents: write`
  only to the `publish` job; `--locked` on the native build.
- `build-all.sh`: `--locked`.

### Flagged, not changed (design / parity — need a maintainer decision)
- **Mutable key material by reference**: `KeyPair.PublicKey/PrivateKey`,
  `PublicKeyReference.PublicKey`, `StoredKeyInfo.PublicKey` expose the internal `byte[]`; a caller
  can mutate it. Migrated verbatim; cloning changes valid-input behavior + perf. Decide:
  defensive copy vs `ReadOnlyMemory<byte>` vs documented no-mutation contract.
- **`StoredKeyInfo` record equality**: `record` + `byte[]` ⇒ *reference* equality, so value-equal
  infos are unequal (footgun for HashSet/dedup). May be intended identity semantics; left as-is.
- **`DefaultKeyGenerator.FromPublicKey`**: no length/curve validation (garbage-in-garbage-out, no
  crash). Adding it would change accepted-input semantics — confirm intent first.
- **CI residuals**: `actions/*` pinned to `@v4` (mutable) not SHA; SHA256SUMS is descriptive,
  not a pre-publish gate; no `persist-credentials: false`. Lower-value given perms are now scoped.
